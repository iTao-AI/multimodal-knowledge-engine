# Contributing

1. Read `AGENTS.md`, the affected ADRs, and relevant tests.
2. Create a short `codex/<scope>-<slug>` branch.
3. Write or update a failing test before changing behavior.
4. Keep documentation in the same PR as behavior.
5. Run the verification commands that exist in the repository.
6. Record actual results and remaining risks in the PR.

Do not push, create a PR, merge, release, or publish without explicit authorization.
