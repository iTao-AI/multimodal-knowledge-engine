# v0.1.4

## Result

Bounded direct audio extends the existing local-first Evidence lifecycle to MP3, WAV/PCM, and
M4A/AAC clips up to 15 minutes and 100 MiB. On an explicitly supervised Darwin arm64 owner, the
cache-only faster-whisper path is shared by Python, CLI, and stdio MCP. Successful Runs publish
timestamp Evidence to an active Publication for Search/Ask and Compiled Library Export v2.

The release keeps the existing proof, demo, CLI, MCP, local knowledge proof, and retrieval
evaluation surfaces and retrieval evaluation documentation. Use [Verify The
Release](../how-to/verify-release.md), [Use bounded direct
audio](../how-to/use-direct-audio.md), [Run the direct-audio
proof](../how-to/run-direct-audio-proof.md), and [Run The Compiled Library Export
Proof](../how-to/run-compiled-library-export-proof.md) for the bounded verification paths.

## Compatibility And Authority

- Export v1 remains compatible for PDF/video-only Libraries. Export v2 adds explicit audio media
  types and the portable `mke.audio_transcript.v1` artifact without weakening v1 validation.
- SQLite remains domain truth. Assets and Artifacts remain immutable, and Search/Ask read only
  active Publication Evidence.
- `cjk-active-scan-overlap-v1` remains the runtime default.
- Dense, RRF, relevance-gate, and reranker records remain comparison-only evaluation evidence.
- LLM Wiki compatibility remains isolated external downstream evidence. MKE does not bundle or
  automatically integrate LLM Wiki, and LLM Wiki is not an MKE runtime dependency or Evidence
  authority.
- The accepted dependency receipt records Torch and Setuptools licensing and dependency
  identities. MKE does not redistribute external wheels or native binaries.

## Explicit Non-Claims

- No implicit model or wheel download, cloud transcription fallback, arbitrary audio/container
  support, long-audio workflow, transcript-accuracy guarantee, production SLA, or cross-platform
  support is claimed.
- Production OCR remains excluded. OCR remains evaluation-only frozen authority.
- There is no PyPI or package-registry publication and no hosted deployment.
- MKE is not deployed in production and does not claim production adoption, real-user outcomes,
  or business impact.
- The release boundary permits zero extra assets; candidate state is not publication authority.
