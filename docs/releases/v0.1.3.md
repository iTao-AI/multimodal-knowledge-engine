# v0.1.3 Release Notes

`v0.1.3` adds Compiled Library Export while preserving the local-first Evidence engine, active
Publication authority, and `cjk-active-scan-overlap-v1` runtime default.

## Compiled Library Export

`mke library export` reads one active-Publication SQLite snapshot without mutation and publishes a
new compiled Library directory through a descriptor-bound, manifest-last transaction. The canonical
manifest uses `mke.compiled_library_export.v1`; Markdown files use
`mke.compiled_markdown.v1`; JSONL records use authoritative `mke.evidence_ref.v1` provenance.

The Markdown is a readable derivative. EvidenceRef JSONL remains the machine authority for exact
page or timestamp Evidence. An installed-wheel external consumer proof validates deterministic
producer bytes and portable consumption with the same wheel on Python 3.12 and Python 3.13.

## PDF OCR Phase 0 Evidence

PDF OCR Phase 0 is bounded closed-protocol planning evidence on a fixed synthetic corpus.
PP-OCRv6 medium is the selected production-planning baseline. PaddleOCR-VL 1.6 is a validated
comparison candidate. These observations are not production OCR, a public OCR runtime, provider
promotion, a general quality claim, a production resource limit, or layout reconstruction.

## Verification

The release candidate is gated by full Pytest, Ruff, Pyright, build, `mke proof run`,
`mke demo --verify`, the local knowledge proof, Evidence provenance proof, presentation audit,
installed-wheel consumer smoke, consumer source-pack proof, retrieval evaluation validators, and
the Compiled Library Export proof.

The exact current wheel command is:

```bash
UV_OFFLINE=1 uv run python scripts/release_consumer_smoke.py \
  --wheel dist/multimodal_knowledge_engine-0.1.3-py3-none-any.whl --json
```

## Boundaries

LLM Wiki compatibility is deferred to a separate isolated acceptance plan.
Production OCR remains excluded.
Dense, RRF, and reranker artifacts remain comparison-only evidence and are not runtime promotion.
No hosted integration is claimed. HTTP/UI, PyPI publication, deployment, production adoption, and
business-impact claims remain excluded.
