# Local Knowledge Proof Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this
> plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a deterministic public-safe stdio MCP proof for two synthetic local PDF knowledge
files without changing runtime contracts.

**Architecture:** A reproducible fixture generator owns the two committed PDFs and their manifest.
A proof module drives the real stdio MCP server through the official Python SDK, retains transient
identifiers only in memory, validates ingest/Run/Search/Ask/refusal behavior, and emits a redacted
aggregate report. A thin script is the documented entrypoint.

**Tech Stack:** Python 3.12/3.13, PyMuPDF, MCP Python SDK, SQLite, pytest, Ruff, Pyright, uv.

## Global Constraints

- Keep `mke proof run` at its existing eight cases and do not change CLI or MCP schemas.
- Do not add dense, RRF, reranker, HTTP, UI, OCR, API-adapter, or model runtime behavior.
- Use only repository-authored synthetic fixtures; do not access private files or the network.
- Never render temporary paths, transient IDs, Evidence text, subprocess details, or tracebacks.
- Do not change version metadata or CHANGELOG and do not push, create a PR, tag, or release.

### Authoritative P1 Findings

1. Search and Ask Evidence validation checked only shape and count. Unrelated content, plausible
   same-keyword fabricated content, negative page starts, and reversed page ranges could pass the
   proof.
2. Published Run inspection required only a non-empty events list. Malformed event entries,
   discontinuous indices, missing lifecycle stages, and post-activation events could pass.

### Remediation

- Evidence validation now binds Search and Ask results to the fixed synthetic fixture content by
  SHA-256, retains query-term validation, and requires page locators with `start >= 1` and
  `end >= start`.
- Run validation now requires the public event shape, contiguous indices beginning at one, the
  required lifecycle in order, and final `publication_activated`.
- Adversarial RED tests were added before each minimal implementation change. No MCP contract,
  runtime, fixture text, or proof output structure changed.

  command; historical observations were unchanged.
